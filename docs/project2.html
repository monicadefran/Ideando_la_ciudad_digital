Objetivo:

El objetivo principal de este proyecto es mostrar los conceptos más básicos de la herramienta App Inventor al mismo tiempo que se desarrolla un proyecto.

Conceptos Básicos:

  • Interfaz de Usuario: Es el medio por el que los usuarios se pueden comunicar con las máquinas, computadoras o dispositivos y viceversa. Ejemplo: Pantalla de móvil, teclado de ordenador, altavoz, micrófono…
  • Convertir el código fuente (texto “entendible” por el ser humano) u otra representación de lógica/acciones en un código comprensible por un ordenador.

Pasos a Seguir

  1. Abrir el entorno de desarrollo App Inventor.
    1. http://appinventor.mit.edu/explore/#
    2. Pulsar el botón create app!
    3. Selecciona una cuenta de Google
  2. Pulsa el botón “Start new proyect”
    3. Una vez creado el proyecto podremos acceder a la interfaz de App Inventor que está compuesta por dos pestañas principales: Designer y Blocks. Designer permite diseñar la interfaz de la aplicación que se desarrolla arrastrando los componentes desde el margen izquierdo al Viewer. Las propiedades de los componentes pueden ser modificados en el margen derecho. Blocks permite definir la lógica de la aplicación mediante bloques. Para ello, basta con mover los componentes del margen derecho que intervienen en la lógica al lienzo y describir la lógica.
  3. Arrastra el componente Label del margen derecho (Palette->User Interface) al visor. Cambia las siguientes propiedades del componente:
    1. FontBold: “marcado” (hará que el texto esté en negrita)
    2. FontSize: 28 (hará que el texto sea más grande)
    3. Width: “Fill parent” (hará que ocupe todo el ancho de la pantalla)
    4. TextAligment: “Center: 1” (hará que el texto esté centrado)
    5. Text: “Welcome UC3M!” (cambiará el texto que se muestra)
  4. Arrastra el componente VerticalArrangement del margen derecho (Palette-> Layout) debajo de la etiqueta anterior. Este componente permite colocar los elementos horizontalmente. Cambia las siguientes propiedades del componente:

  5. Arrastra el componente Label y TextBox (Palette->User Interface) dentro del componente anterior. Cambia las siguientes propiedades de los componentes:

    15. Explicación General del Código

    La lógica de todos sketches o programas se divide en dos grandes partes (como todos los sketches para Arduino).

    • Función Setup: Se ejecuta el primero y una única vez
    • Función Loop: Se ejecuta de manera infinita en forma de bucle después de Setup

    Explicación del Código de este Ejemplo

    En la función Setup aparece una única línea que configura la velocidad a la que se van a comunicar la placa y el ordenador. Puede ser cambiada a cualquier valor válido (300, 1200, 2400, 4800, 9600, 19200, 38400, 57600, 74880, 115200, 230400, 250000). Si se desea cambiarlo habría que indicar en la consola de monitorización la misma velocidad para que se haga correctamente.

    En la función Loop aparecen dos líneas. La primera escribe por el cable serial (el cable usb) el mensaje que aparece. La segunda línea indica a la placa que debe de esperar 3000 milisegundos que es igual a 3 segundos. Tras terminar de ejecutar esta línea de código se vuelve e ejecutar de nuevo la primera línea por eso aparece cada tres segundos un mensaje por consola.
